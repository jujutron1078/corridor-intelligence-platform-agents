# Economic Impact Modeling Agent

This agent models the macroeconomic and development impact of corridor infrastructure investments. It quantifies GDP effects, employment, poverty reduction, catalytic sector unlock, and regional integration, and compares long-term scenarios with and without the project.

---

## Shared Tools (Platform-Wide)

All agents in the platform use these common tools for reasoning and task management:

| Tool | Purpose |
|------|--------|
| **think_tool** | Allows the agent to reason step-by-step, weigh options, and document assumptions before taking action. |
| **write_todos** | Lets the agent create and update a structured task list for multi-step workflows. |

## Domain Tools

### 1. `calculate_gdp_multipliers`

**What it does:** Calculates the **direct, indirect, and induced GDP impacts** of the infrastructure investment. It uses Input-Output (I-O) models to determine how construction spending and industrial enablement ripple through the regional economy.

**When to use:** When you need to quantify total economic output generated by the project (beyond direct construction spend), e.g. for DFI reporting or government business cases.

**Input:**

| Field | Type | Description |
|-------|------|-------------|
| `total_capex` | float | Total transmission investment from the Financial Agent. |
| `industrial_output` | float | Estimated annual production value from anchor loads. |
| `region_io_model` | str | The Input-Output model for the corridor (default: `"West_Africa_2024_IO"`). |

**Output:** JSON with analysis header, methodology, multiplier structure, consolidated GDP impact, and supporting detail:

| Section | Contents |
|---------|----------|
| `corridor_id`, `analysis_type`, `analysis_scope`, `analysis_horizon_years`, `currency` | Analysis header. |
| `methodology` | Model type, reference frameworks, multiplier basis, domestic value capture, computation mode. |
| `multiplier_structure` | `direct_effect_multiplier`, `indirect_effect_multiplier`, `induced_effect_multiplier`, `total_regional_multiplier`, benchmark range. |
| `gdp_impact_summary` | `gross_capex_reference_usd`, `total_gdp_impact_usd`, `average_annual_gdp_impact_usd`, `peak_year_impact_usd`, `share_of_regional_gdp_pct`. |
| `impact_breakdown_usd` | `direct_effects` (construction_wages, engineering_services, local_material_supply), `indirect_effects`, `induced_effects` with subtotals. |
| `impact_by_country` | Per-country `gdp_impact_usd`, `pct_of_total`, `primary_drivers` (e.g. Côte d'Ivoire, Ghana, Togo, Benin, Nigeria). |
| `sectoral_gdp_contribution` | Share by sector (construction, manufacturing, transport, services, energy). |
| `macroeconomic_transmission_channels` | Investment, consumption, capacity utilization, cross-border channels. |
| `sensitivity_band_analysis` | `low_case_multiplier`, `high_case_multiplier`, `low_case_total_gdp_usd`, `high_case_total_gdp_usd`, `variance_driver`. |
| `audit_metadata` | Version, generator, timestamp, data source, validation status, intended use. |
| `message` | Short executive summary. |

**Key concepts:** Direct impact (construction/operations spend), indirect (supply chain), induced (household spending from new income). Output is typically expressed as multipliers or total GDP delta.

### 2. `model_employment_impact`

**What it does:** Estimates **job creation** across construction, operations, and enabled industries. It differentiates between direct infrastructure roles (construction, O&M) and secondary jobs created in sectors like mining and manufacturing.

**When to use:** To report jobs created (FTE or person-years), by phase and by sector, for feasibility studies and development impact narratives.

**Input:**

| Field | Type | Description |
|-------|------|-------------|
| `anchor_load_portfolio` | list[dict] | Portfolio from Agent 2 (Opportunity Identification). |
| `construction_duration_years` | int | Build period in years (default: 5). |

**Output:** JSON with employment summary, breakdowns, country distribution, skills/inclusion, time profile, and sensitivity:

| Section | Contents |
|---------|----------|
| `corridor_id`, `analysis_type`, `analysis_scope`, `analysis_horizon_years`, `construction_period_years`, `currency` | Analysis header. |
| `methodology` | Computation mode, job accounting units (construction/operations/enabled/induced), definitions, core benchmarks, key assumptions (local labor share, procurement, enabled jobs realization rate), important notes. |
| `employment_summary` | `direct_construction_jobs`, `direct_operational_jobs`, `enabled_industrial_jobs`, `induced_jobs`. |
| `total_jobs_created` | Aggregate job count. |
| `breakdowns` | `direct_construction_detail` (peak_headcount_jobs, total_person_years, time_profile_peak_years, job_families), `direct_operations_detail` (ongoing_FTE_jobs, operations_job_families), `enabled_industrial_detail` (enabled_job_sources, anchor_load_dependency, ramp_profile), `induced_detail` (induced_job_sources, income_transmission_logic). |
| `impact_by_country` | Per-country share_pct, direct_construction_jobs, direct_operational_jobs, enabled_industrial_jobs, induced_jobs, primary_drivers. |
| `skills_and_inclusion` | construction_skill_split_pct, operations_skill_split_pct, localization_and_capacity_building (training_required_roles, recommended_training_programs), gender_lens. |
| `time_profile` | construction_phase_peak_jobs_by_year, operations_phase_FTE_by_year, enabled_jobs_realization_path, interpretation. |
| `sensitivity_band_analysis` | low_case, base_case, high_case (enabled_jobs_realization_rate, total_jobs_created, main drivers). |
| `audit_metadata` | Version, generator, timestamp, data source, validation status, intended use. |
| `message` | Short executive summary. |

**Key concepts:** Direct jobs (building and operating the asset), indirect (supply chain), induced (from increased household income). Often broken down by skill level and gender where data allows.

### 3. `assess_poverty_reduction`

**What it does:** Models the impact of **electricity access expansion on household income**. It calculates how many people are expected to be lifted above the poverty line due to new energy access and associated income effects.

**When to use:** When you need SDG-aligned metrics (e.g. people brought above poverty line) or household-level welfare impact for development reporting.

**Input:**

| Field | Type | Description |
|-------|------|-------------|
| `baseline_electrification_rate` | float | Current access rate in the zone. |
| `projected_beneficiaries` | int | Estimated people gaining access. |

**Output:** JSON with poverty impact, welfare detail, country distribution, time profile, and sensitivity:

| Section | Contents |
|---------|----------|
| `corridor_id`, `analysis_type`, `analysis_scope`, `analysis_horizon_years`, `currency` | Analysis header. |
| `methodology` | Computation mode, poverty_framework (primary line $2.15/day, secondary lines), modeled population segments, impact_channels, core_assumptions (household size, poverty rate, income uplift %, energy savings, transition rate). |
| `poverty_reduction_impact` | `additional_people_electrified`, `estimated_poverty_reduction_count`, `avg_household_income_gain`. |
| `welfare_and_poverty_detail` | `affected_population` (newly_electrified_people, reliability_upgraded_people, households), `poverty_reduction_breakdown` (people_lifted_above_2_15_line, households_lifted, people_near_threshold), `income_and_cost_effects` (average_household_income_uplift, energy_cost_savings), `pathways_to_income_gain` (productive_use_examples, reliability_examples). |
| `impact_by_country` | Per-country `additional_people_electrified`, `people_lifted_above_2_15_line`, `primary_drivers`. |
| `time_profile` | electrification_rollout_people by year, poverty_transitions_people by year, interpretation. |
| `sensitivity_band_analysis` | low_case, base_case, high_case (people_lifted_above_2_15_line, assumptions), main_variance_drivers. |
| `audit_metadata` | Version, generator, timestamp, data source, validation status, intended use. |
| `message` | Short executive summary. |

**Key concepts:** Links electrification to income (productivity, new enterprises, reduced energy costs) and applies poverty line thresholds to estimate headcount impact.

### 4. `quantify_catalytic_effects`

**What it does:** Quantifies how infrastructure **unlocks specific sectors**. It calculates the "Unlock Value" for mining, agriculture, and manufacturing by modeling reduced OPEX and energy costs enabled by the corridor.

**When to use:** To show which industries benefit most from the project and by how much (e.g. $M in additional sector output or cost savings).

**Input:**

| Field | Type | Description |
|-------|------|-------------|
| `sector_growth_targets` | dict[str, float] | Growth goals for mining, agriculture, etc. |

**Output:** JSON with sector unlock values and digital economy impact:

| Section | Contents |
|---------|----------|
| `sector_unlock_value_usd` | Unlock value by sector: `agriculture_agro_processing`, `mining_minerals`, `manufacturing` (e.g. "1.2B", "2.4B", "0.9B"). |
| `digital_economy_impact` | Qualitative assessment (e.g. "High (Fiber-optic backbone included)"). |
| `message` | Short confirmation (e.g. "Sector-specific catalytic effects quantified."). |

**Key concepts:** Catalytic effect = economic value created in other sectors because of the new infrastructure (e.g. cheaper power → more viable mining/agro-processing).

---

### 5. `model_regional_integration`

**What it does:** Models **cross-border trade and market integration** impacts. It calculates the increase in regional trade volume within the relevant bloc (e.g. ECOWAS) due to improved connectivity from the corridor.

**When to use:** For regional integration narratives, trade facilitation metrics, and multi-country project justification.

**Input:**

| Field | Type | Description |
|-------|------|-------------|
| `trade_volume_baseline` | float | Current intra-regional trade value. |
| `transport_cost_reduction` | float | Expected cost reduction (default: 0.25). |

**Output:** JSON with integration score and trade projection:

| Section | Contents |
|---------|----------|
| `regional_integration_score` | Numeric score (e.g. 0.88). |
| `projected_trade_volume_increase` | Percentage increase (e.g. "32%"). |
| `key_benefit` | Short narrative (e.g. "Enhanced energy security and synchronized industrial standards."). |
| `message` | Short confirmation (e.g. "Regional integration benefits projected for ECOWAS members."). |

**Key concepts:** Connects transport/energy connectivity to trade flows, tariff reduction potential, and formalization of informal cross-border trade.

---

### 6. `perform_impact_scenario_analysis`

**What it does:** Compares **"Business as Usual" vs. "Enhanced Development"** trajectories. It provides a 20-year delta analysis of the economy with and without the infrastructure.

**When to use:** To produce the headline counterfactual: how much better is the economy (GDP, jobs, poverty, trade) with the project than without it.

**Input:**

| Field | Type | Description |
|-------|------|-------------|
| `baseline_growth_rate` | float | Projected GDP growth without project. |
| `time_horizon_years` | int | Analysis horizon in years (default: 20). |

**Output:** JSON with scenario comparison and confidence:

| Section | Contents |
|---------|----------|
| `scenario_comparison` | `baseline_20_yr_gdp_growth`, `enhanced_20_yr_gdp_growth`, `total_value_add_usd` (e.g. "7.5B annually by Year 10"). |
| `confidence_interval` | e.g. "95%". |
| `message` | Short confirmation (e.g. "Comparative impact analysis complete."). |

**Key concepts:** Baseline scenario (no project) vs. project scenario; outputs are typically deltas (e.g. additional GDP, jobs, people out of poverty) over the analysis horizon.

---

## Typical Workflow

1. **Inputs:** Receives corridor scope, CAPEX/OPEX, and demand/load assumptions from other agents (e.g. Infrastructure, Opportunity Identification).
2. **GDP & jobs:** Use `calculate_gdp_multipliers` and `model_employment_impact` for headline macro metrics.
3. **Development depth:** Use `assess_poverty_reduction`, `quantify_catalytic_effects`, and `model_regional_integration` for sector and welfare impact.
4. **Scenario wrap-up:** Use `perform_impact_scenario_analysis` to compare with/without and produce a consolidated impact narrative for stakeholders and DFIs.
